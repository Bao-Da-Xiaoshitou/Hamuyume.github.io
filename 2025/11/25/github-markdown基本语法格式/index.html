基本写作和格式语法

使用简单的语法在 GitHub 上为您的散文和代码创建复杂的格式

标题的创建

如果想要创建标题，请在标题文本前添加一到六个#符号。使用的#数量将决定标题的层级和字体大小

当您使用两个或更多标题时，GitHub 会自动生成一个目录，您可以点击文件标题中的链接访问。每个标题都列在目录中，您可以点击标题导航到选定的部分。

文本的样式

您可以在评论字段和.md文件中使用粗体、斜体、删除线、下标或上标文本来表示强调。

样式 语法 快捷键 示例 输出
粗体 ** ** 或 __ __ Command+B (Mac) 或 Ctrl+B (Windows/Linux) **这是粗体文本** 这是粗体文本
斜体 * * 或 _ _ Command+I (Mac) 或 Ctrl+I (Windows/Linux) _这段文字是斜体的_ 这段文字是斜体的
删除线 ~~ ~~ 无 ~~这是错误的文本~~ 这是错误的文本
粗体和嵌套斜体 ** ** 和 _ _ 无 **这段文字_非常_重要** 这段文字_非常_重要
全部粗体和斜体 *** *** 无 ***所有这些文字都很重要*** 所有这些文字都很重要
下标 <sub> </sub> 无 这是一个<sub>下标</sub>文本 这是一个下标文本
上标 <sup> </sup> 无 这是一个<sup>上标</sup>文本 这是一个上标文本
下划线 <ins> </ins> 无 这是一个<ins>下划线</ins>文本 这段文字带有下划线

文本的引用

您可以使用>来引用文本。

引用文本缩进，并使用不同的字体颜色。

代码的引用

您可以使用单个反引号来在句子中指出代码或命令。反引号内的文本不会被格式化。您也可以按Command+E (Mac) 或 Ctrl+E (Windows/Linux) 快捷键，在 Markdown 行中插入代码块的反引号。

要将代码或文本格式化为其自身的独立块，请使用三个反引号。
